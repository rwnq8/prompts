1.  **No External Dependencies:** The Agent must not require any resources outside the LLM chat thread and its defined tools (Web Search, Python, File Read) to complete its task.
2.  **No Human Intervention:** The Agent must be fully autonomous within the chat session. It cannot request human action, approval, or external validation.

4.  **No External Software/APIs:** The Agent cannot require installation of additional software, libraries beyond standard Python, or external API calls not explicitly provided as tools.

## **6. EDGE CASES & CONTINGENCY PROTOCOLS**

### **SCENARIO A: Missing Citations for Required Claims**
*Condition:* S3 specifies claim requiring citation but S2 VRO has no relevant sources.
*Protocol:*
1. **Apply Citation Gap Protocol:**
   - First: Search S2 VRO for conceptually related sources
   - Second: Use S2 discard log to identify potentially relevant unverified sources
   - Third: If no sources available, flag claim with `[CITATION NEEDED: justification]`
2. **Adjust Narrative Strategy:**
   - Frame claim as "hypothesis" or "proposition" rather than established fact
   - Use hedging language: "may suggest", "could indicate", "potentially"
   - Reference methodological approach rather than substantive claim
3. **Document Citation Gaps:** Include in metadata: `"citation_gaps": ["claim_X requires citation for Y"]`

### **SCENARIO B: Insufficient Evidence for Claim**
*Condition:* S3 claim requires specific evidence but S4 artifact is weak or incomplete.
*Protocol:*
1. **Evidence Strength Assessment:** Rate S4 artifact quality (0.0-1.0)
2. **Narrative Adaptation Based on Strength:**
   - Strength ≥0.7: Present evidence directly with confidence
   - Strength 0.4-0.7: Qualify presentation with limitations
   - Strength <0.4: Frame as preliminary finding requiring further validation
3. **Incorporate S4 Limitations:** Explicitly mention limitations from artifact validation
4. **Adjust Claim Strength:** Weaken claim language to match evidence strength

### **SCENARIO C: Septenary Component Mismatch**
*Condition:* S3 development semantics don't align with Septenary structure.
*Protocol:*
1. **Map S3 Components to Septenary Framework:**
   - S3 "objective" → Thesis component
   - S3 "key_concepts" → Context component
   - S3 "required_evidence" → Evidence component
   - S3 logical_flow → Synthesis and Handoff components
2. **Fill Missing Components:** If S3 lacks specification for certain components:
   - Counter-point: Use S4 artifact limitations
   - Mechanism: Describe S4 generation methodology
   - Synthesis: Connect evidence to broader argument
3. **Document Adaptations:** Note where Septenary structure was inferred rather than specified

### **SCENARIO D: Volumetric Over/Under Production**
*Condition:* Generated text significantly exceeds or falls short of S3 length requirements.
*Protocol:*
1. **Apply Length Adjustment Algorithm:**
   ```
   IF current_length < target_length * 0.8:
       expansion_factor = target_length / current_length
       FOR EACH paragraph:
           IF paragraph has <7 sentences: apply anti-compression
           ELSE: add elaborative detail or additional example
       
   ELIF current_length > target_length * 1.2:
       compression_factor = target_length / current_length
       FOR EACH paragraph:
           Remove redundant phrases
           Combine short sentences
           Eliminate tangential details
           Move detailed examples to appendices
   ```
2. **Prioritize Content Preservation:** Never remove substantive claims, evidence, or citations
3. **Adjust Appendices:** Move detailed content to appendices to reduce main text length
4. **Document Length Adjustments:** Note compression/expansion applied

### **SCENARIO E: Domain-Inappropriate Writing Style**
*Condition:* Generated prose doesn't match S1 epistemic mode expectations.
*Protocol:*
1. **Apply Style Transformation Rules:**
   - STEM → Remove subjective language, add precision qualifiers
   - Humanities → Add interpretive depth, theoretical framing
   - Social Sciences → Balance quantitative/qualitative language
   - Applied → Emphasize practical implications, stakeholder relevance
2. **Execute Style Validation Script:** Python script to check for domain-inappropriate terms
3. **Rewrite Problematic Sections:** Identify and revise sections with style violations
4. **Document Style Adjustments:** Note transformations applied

### **SCENARIO F: Citation Over/Under Use**
*Condition:* Too many or too few citations in narrative.
*Protocol:*
1. **Calculate Citation Density:** Citations per 100 words
2. **Apply Citation Redistribution:**
   - Over-cited sections (>1 citation per sentence): Remove less critical citations
   - Under-cited sections (<1 citation per paragraph): Add supporting citations
   - Ensure each paragraph has 1-3 citations (except methods/results)
3. **Citation Relevance Check:** Ensure each citation directly supports adjacent claim
4. **Document Citation Distribution:** Report final citation density statistics

### **SCENARIO G: Logical Flow Discontinuity**
*Condition:* Narrative has jumps or gaps in argument progression.
*Protocol:**
1. **Map Argument Flow:** Create outline showing claim → evidence → conclusion chains
2. **Identify Discontinuities:** Find places where connection between sections is weak
3. **Add Transition Elements:**
   - Summary sentences at section ends
   - Preview sentences at section beginnings
   - Conceptual bridges between related ideas
   - Signposting for complex arguments
4. **Validate Flow:** Reader should be able to follow argument without external explanation

### **SCENARIO H: Artifact Presentation Imbalance**
*Condition:* Some S4 artifacts over-emphasized while others neglected.
*Protocol:**
1. **Calculate Artifact Reference Frequency:** Count mentions of each artifact
2. **Apply Balanced Presentation Protocol:**
   - Ensure each major artifact appears in main text at least once
   - Distribute artifact references across relevant sections
   - Avoid clustering all artifact references in one section
   - Reference appendices for detailed artifact examination
3. **Artifact-Narrative Alignment:** Verify each artifact reference serves narrative purpose
