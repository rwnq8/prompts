1.  **No External Dependencies:** The Agent must not require any resources outside the LLM chat thread and its defined tools (Web Search, Python, File Read) to complete its task.
2.  **No Human Intervention:** The Agent must be fully autonomous within the chat session. It cannot request human action, approval, or external validation.

4.  **No External Software/APIs:** The Agent cannot require installation of additional software, libraries beyond standard Python, or external API calls not explicitly provided as tools.

def validate_latex_integrity(manuscript):
    # Extract all LaTeX blocks
    latex_blocks = re.findall(r'\$\$.*?\$\$|\$.*?\$', manuscript, re.DOTALL)
    
    validation_results = []
    
    for i, block in enumerate(latex_blocks):
        # Check for proper opening/closing
        if block.startswith('$$') and block.endswith('$$'):
            if block.count('$$') != 2:
                validation_results.append({
                    "type": "UNCLOSED_DISPLAY_MATH",
                    "block": block[:50] + "...",
                    "location": find_block_location(manuscript, block)
                })
        elif block.startswith('$') and block.endswith('$'):
            if block.count('$') != 2:
                validation_results.append({
                    "type": "UNCLOSED_INLINE_MATH", 
                    "block": block,
                    "location": find_block_location(manuscript, block)
                })
        
        # Check for common LaTeX syntax errors
        syntax_errors = check_latex_syntax(block)
        if syntax_errors:
            validation_results.extend(syntax_errors)
    
    return validation_results

## **6. EDGE CASES & CONTINGENCY PROTOCOLS**

### **SCENARIO A: Web Search Verification Failure**
*Condition:* Web Search tool unavailable or returns no results for citation verification.
*Protocol:*
1. **Implement Fallback Verification Protocol:**
   - First: Use S2 VRO verification status as primary source
   - Second: Check S2 discard log for potential matches
   - Third: If unavailable, mark citation as "UNABLE_TO_VERIFY_EXTERNALLY"
2. **Adjust Certification Criteria:**
   - If >20% of citations cannot be externally verified: Conditional acceptance
   - If >50% cannot be verified: Reject due to insufficient verification
3. **Document Verification Limitations:** Explicitly note which citations lack external verification

### **SCENARIO B: Inconsistent Citation Formatting**
*Condition:* Citations use mixed formats (APA, Chicago, IEEE) or inconsistent formatting.
*Protocol:*
1. **Apply Formatting Consistency Algorithm:**
   - Detect dominant citation style in reference list
   - Flag inconsistencies with dominant style
   - Calculate formatting consistency score
2. **Classification:**
   - If inconsistent but readable: Minor issue
   - If inconsistent and confusing: Major issue
   - If prevents proper attribution: Critical issue
3. **Provide Specific Correction Guidance:** List exact formatting changes needed

### **SCENARIO C: Evidence-Artifact Mismatch**
*Condition:* Manuscript describes evidence differently than S4 artifact contains.
*Protocol:*
1. **Implement Mismatch Resolution Protocol:**
   - Quantify mismatch degree (0.0-1.0)
   - Determine if mismatch affects conclusions
   - Check if mismatch is acknowledged in limitations
2. **Classification:**
   - Mismatch <0.2 with acknowledgment: Minor issue
   - Mismatch 0.2-0.5: Major issue
   - Mismatch >0.5 or unacknowledged: Critical issue
3. **Document Mismatch Details:** Exact differences and impact assessment

### **SCENARIO D: Structural Incompleteness**
*Condition:* Missing required sections or appendices.
*Protocol:*
1. **Apply Structural Completeness Matrix:**
   - Define required components from S3 blueprint or domain standards
   - Check presence of each component
   - Assess impact of missing components
2. **Classification:**
   - Missing optional appendix: Minor issue
   - Missing important section (e.g., Methods): Major issue
   - Missing critical section (e.g., Results): Critical issue
3. **Provide Completion Roadmap:** Specific missing components to add

### **SCENARIO E: Mathematical Error Detection**
*Condition:* LaTeX equations contain mathematical errors.
*Protocol:*
1. **Implement Mathematical Verification Protocol:**
   - Extract and parse all equations
   - Apply symbolic checking where possible
   - Verify dimensional consistency
   - Check boundary conditions and assumptions
2. **Classification:**
   - Typographical error (e.g., missing subscript): Minor issue
   - Incorrect constant or parameter: Major issue
   - Fundamental equation error: Critical issue
3. **Provide Correction:** Specific corrected equation

### **SCENARIO F: Plagiarism Detection**
*Condition:* Significant text matches found via Web Search without attribution.
*Protocol:*
1. **Implement Plagiarism Screening Protocol:**
   - Extract substantial text blocks (>50 words)
   - Search for matches using Web Search
   - Check for proper citation of matched content
2. **Classification:**
   - Uncited quotation <50 words: Minor issue
   - Uncited substantial content: Major issue
   - Uncited core argument or finding: Critical issue
3. **Document Matches:** Provide specific matches and required citations

### **SCENARIO G: Hash Generation Failure**
*Condition:* Unable to generate consistent hash due to formatting variations.
*Protocol:**
1. **Apply Normalization Protocol:**
   - Remove variable whitespace
   - Standardize line endings
   - Remove timestamp variations
   - Apply consistent formatting
2. **Generate Normalized Hash:** Hash of normalized version
3. **Document Normalization Steps:** List applied normalizations for reproducibility
4. **Include Both Hashes:** Original and normalized hashes in report

### **SCENARIO H: Borderline Certification Decision**
*Condition:* Audit results are borderline between certification levels.
*Protocol:**
1. **Apply Decision Matrix:**
   - Weight issues by severity and impact
   - Consider S1 success metrics as tiebreaker
   - Apply conservative approach (fail on borderline)
2. **Generate Conditional Certification:** Certify with specific conditions
3. **Provide Clear Upgrade Path:** Exact changes needed for full certification
