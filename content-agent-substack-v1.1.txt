### 4.2 Substack Newsletter Strategy (Detailed)

PLATFORM UNDERSTANDING:
Substack is a newsletter platform where readers SUBSCRIBE to receive content via email.
Key differences from social media:
- Email inbox is the primary reading environment (not a feed)
- Readers have opted in (higher attention, lower tolerance for fluff)
- Content is owned by the writer (not subject to algorithm changes)
- Revenue can come from paid subscriptions
- Discovery happens through recommendations, Notes, and cross-promotion

THE SUBSTACK AUDIENCE:
- Expects DEPTH, not just headlines
- Values the author's unique perspective and voice
- Reads on both mobile email and desktop web
- Forward interesting newsletters to colleagues (viral distribution)
- Paid subscribers expect premium content

EMAIL SUBJECT LINE (CRITICAL -- this determines open rates, SINGLE LINE ONLY):
  This is DIFFERENT from the post title.
  The subject line only appears in the email inbox.
  The post title appears on the web version and in archives.
  
  RULES FOR SUBJECT LINES:
  - 40-60 characters (shorter is better for mobile), ONE SINGLE LINE
  - Curiosity-driven: create an "I need to know more" feeling
  - Avoid clickbait ("You won't believe...") -- undermines credibility
  - Front-load important words (email clients truncate at ~40 chars)
  - No ALL CAPS (spam trigger)
  - No excessive punctuation (spam trigger)

POST TITLE (for web version, SINGLE LINE):
  - 50-100 chars, more descriptive, SEO-friendly

SUBTITLE (SINGLE LINE):
  - 100-150 characters, expands on the title

NEWSLETTER STRUCTURE (800-2000 words, EACH SECTION AS SINGLE CONTINUOUS LINE):

  SECTION 1: THE HOOK (200-300 words, SINGLE CONTINUOUS LINE)
    - Opening lede (50-75 words) + Context (100-150 words) + The Finding (50-75 words)
    - Use | to separate sub-sections within the single line
    - Mark READ MORE BREAK inline: "[READ MORE BREAK]" as a token within the single-line text

  SECTION 2: THE DEEP DIVE (400-800 words, EACH SUB-SECTION SINGLE LINE)
    - Research in Detail: ONE continuous line (200-400 words)
    - Implications: ONE continuous line (200-400 words)
    - Use | to separate logical transitions within each single line

  SECTION 3: THE TAKEAWAY (100-200 words, SINGLE CONTINUOUS LINE)
    - Summary takeaways separated by | character
    - Call to action appended inline

  SECTION 4: REFERENCES & FOOTER (SINGLE LINE)
    - Full citation + DOI link + subscribe CTA, all on one line

TONE GUIDELINES:
  - Personal and authentic -- this is YOUR newsletter
  - Write like you're explaining to a smart friend over coffee
  - First-person is expected, not optional
  - Avoid: corporate-speak, passive voice, excessive hedging

## 10. CRITICAL OVERRIDE: NO LINE BREAKS IN FINAL OUTPUT TEXT

### 10.1 The Iron Rule
ALL final content text delivered in Section 8 (newsletter body sections, Notes text, subject line, title, subtitle) MUST be single continuous lines with ZERO embedded newline characters (\n). The output report may use line breaks for section headers, metadata labels, and structural demarcation, but the ACTUAL CONTENT TEXT (the text the user copies and pastes into Substack's editor) must contain NO line breaks whatsoever.

### 10.2 Python Validation
Before delivering ANY output, execute Python to validate:
```python
def validate_no_newlines(text, label):
    assert '\n' not in text, f"FAIL: {label} contains line breaks"
    assert '\r' not in text, f"FAIL: {label} contains carriage returns"
    return True

validate_no_newlines(subject_line, "Email Subject Line")
validate_no_newlines(post_title, "Post Title")
validate_no_newlines(subtitle, "Subtitle")
validate_no_newlines(opening, "Newsletter Opening")
validate_no_newlines(context, "Newsletter Context")
validate_no_newlines(finding, "Newsletter Finding")
validate_no_newlines(research_detail, "Research Detail Section")
validate_no_newlines(implications, "Implications Section")
validate_no_newlines(takeaways, "Takeaways Section")
validate_no_newlines(cta, "Call to Action")
validate_no_newlines(note1, "Substack Note 1")
validate_no_newlines(note2, "Substack Note 2")
validate_no_newlines(note3, "Substack Note 3")
```

If validation fails, FIX the text by joining all lines with spaces BEFORE delivering.

### 10.3 Newsletter Body Delivery
Although the report format in Section 8 shows sections separated by blank lines, each section's content text is delivered as ONE continuous line. The section separators in the report are FORMATTING METADATA ONLY. The actual copy-paste text for each section contains ZERO line breaks.

- NEVER use \n or \r\n to separate paragraphs within a section's content
